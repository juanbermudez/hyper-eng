MCP Servers Reference

The plugin bundles two MCP servers: Context7 for fetching up-to-date framework documentation, and QFS for fast local file search. For browser testing, use the web-app-debugger agent with Claude Code's built-in Chrome extension.

Known Issue: Auto-Loading

Sometimes the MCP server doesn't load automatically. If Claude can't look up docs, you'll need to add it manually. See Manual Configuration for the fix.

Context7

Ever ask Claude about a framework and get an answer from 2023? Context7 fixes that. It's a documentation service that keeps Claude current with 100+ frameworks—Rails, React, Next.js, Django, whatever you're using. Think of it as having the official docs piped directly into Claude's brain.

Tools Provided

Tool Description
resolve-library-id Maps "Rails" to the actual library identifier Context7 uses. You don't call this—Claude does it automatically
get-library-docs Fetches the actual documentation pages. Ask "How does useEffect work?" and this grabs the latest React docs

QFS (Quick File Search)

QFS is a local file search engine that provides fast BM25 full-text search across your codebase. No cloud services, no API keys—everything runs locally on your machine. Index your projects, workspaces, and documentation for instant search.

Tools Provided

Tool Description
qfs_search BM25 keyword search across indexed files. Fast and accurate full-text search.
qfs_vsearch Vector semantic search (requires embeddings). Find conceptually similar content.
qfs_query Hybrid search combining BM25 and vector with Reciprocal Rank Fusion.
qfs_get Retrieve a specific document by its path. Get full content and metadata.
qfs_status Check index health: collection count, document count, last indexed time.

What Gets Indexed

QFS indexes text-based files in configured collections:

Documents

  • Markdown (.md, .mdx)
  • Text files (.txt)
  • JSON/YAML (.json, .yaml)
  • Config files

Code

  • TypeScript (.ts, .tsx)
  • Rust (.rs)
  • Python (.py)
  • Go, Java, etc.

Example Usage

# Search your indexed files

"Search for authentication handling in the codebase"

"Find all references to the User model"

"Look for error handling patterns"

# QFS searches your local files instantly—no network required

CLI Commands

Manage your search index with the Hypercraft CLI:

# Add a collection to index
hypercraft index add docs ./docs --patterns "**/*.md"

# Build/rebuild the index
hypercraft index build

# Check index status
hypercraft index status

# Search from CLI
hypercraft find "query" --mode bm25 --json

Configuration

{
  "qfs": {
    "type": "stdio",
    "command": "${CLAUDE_PLUGIN_ROOT}/binaries/qfs-mcp",
    "args": [],
    "env": {
      "QFS_DB_PATH": "${HOME}/.cache/hypercraft/qfs/index.sqlite"
    }
  }
}

Index First

QFS requires an index to search. Run hypercraft index add and hypercraft index build before using QFS search tools.

Browser Testing

For browser testing and debugging, use the web-app-debugger agent instead of a separate MCP server. This agent uses Claude Code's built-in Chrome extension (mcp__claude-in-chrome__* tools) for:

  • Taking screenshots - Capture what the user sees
  • Inspecting DOM elements - Read the page structure
  • Reading console logs - Check for JavaScript errors
  • Interacting with UI - Click buttons, fill forms

Example Usage

# Use the web-app-debugger agent for browser verification

/hyper:verify auth-system/task-001

# The agent will:
# 1. Run automated checks (lint, typecheck, test, build)
# 2. Spawn web-app-debugger for manual verification
# 3. Take screenshots and interact with UI
# 4. Report any issues found

No Extra Setup Required

The Chrome extension is built into Claude Code. Just make sure you have the Claude Code extension installed in Chrome and you're good to go.

Manual Configuration

If Context7 doesn't load automatically (you'll know because Claude can't fetch docs), you need to wire it up yourself. It's a quick copy-paste job.

Project-Level Configuration

To enable for just this project, add this to .claude/settings.json in your project root:

{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    },
    "qfs": {
      "type": "stdio",
      "command": "qfs-mcp",
      "env": {
        "QFS_DB_PATH": "~/.cache/hypercraft/qfs/index.sqlite"
      }
    }
  }
}

Global Configuration

Or enable everywhere—every project on your machine gets these servers. Add to ~/.claude/settings.json:

{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    },
    "qfs": {
      "type": "stdio",
      "command": "qfs-mcp",
      "env": {
        "QFS_DB_PATH": "~/.cache/hypercraft/qfs/index.sqlite"
      }
    }
  }
}

Verifying MCP Server

After you add the config, restart Claude Code. Then test that everything works:

# Ask Claude what it has
"What MCP tools do you have access to?"

# Test Context7 (should fetch real docs)
"Look up Rails Active Record documentation"

# If it fails, double-check your JSON syntax and file paths
